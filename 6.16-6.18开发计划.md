# 《红花救援队》6.16 - 6.18 三日冲刺计划

---

## 整体策略

**灰盒优先**：先用纯色方块+文字代替所有美术资源，验证玩法闭环。玩法跑通后再换皮。

**单人可完成**：本计划按 1 名开发者 2 天工作量拆分。如有 2 人，可并行"关卡铺设"与"QTE/UI"。

---

## 6.16（Day 1）：基础框架 + 关卡一核心闭环

### 上午 09:00 - 12:00：项目脚手架 + 网格引擎

| 任务 | 产出 | 时间 |
|------|------|------|
| 初始化项目 | Vite + React + Canvas/Phaser 脚手架 | 30min |
| GridMapSystem | 网格渲染、坐标系统、格子高亮 | 1h |
| 玩家移动 | 点击格子 → 移动到该格（寻路or直线） | 30min |
| TurnSystem | 回合概念、AP=4/回合、回合结束结算 | 1h |

**验证标准**：玩家点格子 → 角色移动 → AP 减少 → AP 归零回合结束 → AP 重置。

---

### 下午 13:00 - 18:00：关卡一全部玩法

| 任务 | 产出 | 时间 |
|------|------|------|
| 关卡一地图数据 | 8x8 JSON：玩家起点/王奶奶/面罩/阀门/障碍/烟雾/火焰/安全区 | 30min |
| Fog of War | 黑色遮罩覆盖未探索格，移动后解锁视野 | 1h |
| 烟雾/火焰格 | 烟雾每回合-5HP，火焰不可通行 | 30min |
| 火势扩散 | 每2回合火焰向外扩散1格 | 30min |
| 障碍物系统 | 冰箱/储物架占格，消耗2AP清除 | 30min |
| 道具系统 | 拾取/使用防烟面罩（视野+1、伤害-80%） | 30min |
| 燃气阀门 | 关闭消耗1AP，关闭后火势不再扩散 | 20min |
| NPCSystem | 王奶奶生命值/恐慌值/信任值数据结构 | 30min |
| 王奶奶随机移动 | 每2回合恐慌移动1格 | 30min |
| 行动卡弹窗 | 4选1按钮，正确/错误反馈 | 30min |
| 安抚流程 | 蹲下说明身份 → 恐慌-30/信任+30 → 解锁跟随 | 20min |
| 撤离跟随 | 王奶奶跟随玩家（Queue延迟3步），到达安全区胜利 | 30min |

**验证标准**：从开局到护送王奶奶到安全区，完整跑通 → 结算面板显示各项得分。

---

### 晚间 19:00 - 21:00（可选）：关卡一润色

| 任务 | 产出 | 时间 |
|------|------|------|
| 关卡一结算面板 | 6项评分 + 总分 + 小红花 | 30min |
| AI复盘接入 | 传入操作记录 → 调用大模型API → 显示复盘文字 | 1h |
| 关卡一连调 | 完整验收：进关卡 → 拿面罩 → 关阀门 → 安抚 → 护送 → 结算 → 复盘 | 30min |

## 6.17（Day 2）：关卡二 + AI Prompt + 部署上线

### 上午 09:00 - 12:00：关卡二全部玩法

| 任务 | 产出 | 时间 |
|------|------|------|
| 关卡二地图数据 | 7x7 JSON：玩家/倒地老人/路人A/路人B/AED柜/围观群众/救护车标记 | 20min |
| 步骤判断流程 | 判断环境→检查意识→派人呼救→派人取AED→判断呼吸，每步行动卡 | 1h |
| 路人A指派 | 点击路人A→弹出指令→选择"拨打120"→启动救护车倒计时 | 20min |
| 路人B指派+NPC移动 | 点击路人B→选择"寻找AED"→路人B每回合移动2格往返AED柜 | 40min |
| 群众占格系统 | 每1-2回合生成占格群众，喊话气泡，堵塞路径 | 30min |
| 疏散群众 | 消耗1AP→清除占格群众 | 10min |
| 回合HP模型 | 每回合-5，未CPR额外-5（详下），错误操作即时扣血 | 20min |

**验证标准**：指派路人A+B后 → 路人B在网格上跑AED → 可以正常到达并触发"交接"。

---

### 下午 13:00 - 17:00：CPR QTE + AED + 部署

| 任务 | 产出 | 时间 |
|------|------|------|
| CPR QTE圆环 | 缩圈动画 + 点击判定（命中/失误）+ 同屏显示网格 | 1.5h |
| QTE-HP挂钩 | ≥70%净扣-1，40-69%净扣-3，<40%净扣-5 | 30min |
| AED交接动画 | 路人B入相邻格→短暂动画→获得AED | 20min |
| AED步骤卡 | 6步顺序点击卡片（开机→贴电极→分析→离开→除颤→继续CPR） | 30min |
| AED效果 | HP+20，后续基础消耗-3/回合 | 15min |
| 关卡二结算 | 7项评分 + 总分 + 小红花 | 30min |
| AI复盘（关卡二） | 接入复盘Prompt | 20min |

**验证标准**：关卡二完整流程 — 步骤判断 → CPR QTE → AED到达 → 使用AED → 救护车到达 → 结算。

---

### 17:00 - 18:30：AI Prompt 批量接入

| 任务 | 说明 | 时间 |
|------|------|------|
| 复盘Prompt（两关共用） | 传入操作记录 → 生成"做对了什么/遗漏了什么/为什么危险/鼓励总结" | 20min |
| 知识卡Prompt（两关共用） | 传入关卡知识点+玩家错误 → 生成知识卡文字 | 15min |
| 王奶奶台词Prompt | 传入恐慌值/信任值 → 生成情绪台词 | 15min |
| 群众台词Prompt | 传入回合数/HP → 生成干扰台词 | 15min |
| 联调+文案检查 | 所有AI输出走关键词白名单过滤 | 20min |

---

### 19:00 - 22:00：主菜单 + 知识卡图鉴 + 部署上线

| 任务 | 产出 | 时间 |
|------|------|------|
| 主菜单 | 开始游戏/关卡选择/知识卡图鉴入口 | 30min |
| 知识卡图鉴 | 通关后解锁的知识卡片展示 | 30min |
| 整体联调 | 主菜单→关卡一→结算→复盘→关卡二→结算→复盘→知识卡 | 40min |
| **部署上线** | 打包 → 部署到 EdgeOne Pages / CloudStudio | 40min |
| 公网链接验证 | 手机/桌面端打开链接，完整走一遍 | 20min |
| 预填AI文案 | 如有AI调用不稳定，准备fallback预生成文案 | 30min |

**部署产物**：一个公网可访问的 H5 链接，包含完整双关卡 + 主菜单 + 知识卡。

## 6.18（Day 3）：测试 + 微调

### 上午 09:00 - 12:00：功能Bug修复

| 测试项 | 关注点 |
|--------|--------|
| 关卡一全流程 | Fog of War解锁/王奶奶移动/面罩视野/阀门关闭/跟随撤离/边界碰撞 |
| 关卡二全流程 | 路人B路径/群众占格堵路/CPR成功率计算/AED步骤顺序/救护车倒计时 |
| 数值边界 | HP归零瞬间、AP=0操作、高HP/低HP结算、回合超限 |
| 重玩 | 关卡结算后重新进入，AP/HP/状态正确重置 |

### 下午 13:00 - 15:00：体验微调

| 调整项 | 说明 |
|--------|------|
| CPR 节奏感 | 圆环收缩速度、命中判定窗口大小是否舒适 |
| 网格视觉反馈 | 烟雾格/火焰格/障碍物格颜色区分是否清晰 |
| 行动卡文案 | 选项是否够迷惑但不反人类 |
| 花花提示时机 | 是否在关键步骤及时弹出，会不会刷屏 |
| AI 文案质量 | 复盘是否跑题、知识卡是否正确 |
| 移动端适配 | 7x7/8x8网格在手机屏幕上是否可点击、QTE区域是否太小 |

### 15:00 - 16:00：最终验收 + 重新部署

| 步骤 | 内容 |
|------|------|
| 修完所有Bug → 重新打包 | 最终版本 |
| 重新部署 | 更新EdgeOne Pages上的版本 |
| 交付物检查 | 公网链接可访问 + 双关卡可通关 + 知识卡可查看 |

## 风险与应对

| 风险 | 概率 | 应对 |
|------|------|------|
| Fog of War 实现卡壳 | 中 | 降级为"全局可见+烟雾格有视觉标识"，可接受 |
| CPR QTE 圆环实现复杂 | 中 | 降级为"节拍条左右移动+点击判定"，核心体验不丢 |
| AI API 不稳定 | 高 | 预生成3-5组固定文案作为fallback |
| 路人B寻路卡死 | 低 | 直接用曼哈顿距离直线移动，不做A*寻路 |
| 部署环境问题 | 中 | 准备EdgeOne Pages和CloudStudio两套方案 |
