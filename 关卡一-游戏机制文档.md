## 关卡一「烟雾厨房里的王奶奶」— 游戏机制全览

本文档基于 `RescueScene.js`（主场景逻辑，约 2400 行）、`RescueUi.js`（UI 渲染层）、`App.jsx`（应用壳与关卡选择）、`GameCanvas.jsx`（Phaser 引擎引导）等源码，逐项提取关卡一中已实现的全部玩法与游戏机制。

---

### 一、核心框架：回合制网格策略

关卡一采用 **回合制 + 网格地图** 的基本框架。玩家在有限回合内，通过消耗行动力执行各类救援行动卡，最终目标是在火场中安抚并护送王奶奶到达安全区。

#### 1.1 网格地图系统

地图为 **10 列 x 8 行** 的网格，每格尺寸 104 x 72 像素，左上角起点坐标 (180, 112)。

格子类型共有 10 种，通过二维数组 `RAW_GRID` 定义初始布局：

| 标记 | 类型 | 说明 | 可通行 |
|---|---|---|---|
| `WALL` | 墙体 | 不可进入的墙壁和建筑外围 | 否 |
| `SAFE` | 安全区 | 楼道出口，胜利判定区域（x<=1, y 在 3~5 之间） | 是 |
| `COR` | 走廊 | 普通走廊地面 | 是 |
| `ENT` | 入口 | 王奶奶家的入户门区域 | 是 |
| `KIT` | 厨房 | 普通厨房地面 | 是 |
| `SMK` | 烟雾 | 低浓度烟雾区域 | 是 |
| `HSMK` | 浓烟 | 高浓度烟雾区域，视觉更暗、伤害更高 | 是 |
| `FIRE` | 火焰 | 明火格，不可通行；动态扩展产生 | 否 |
| `OBS` | 障碍 | 翻倒的冰箱或储物架，需清除后才可通过 | 否（清除后变为 KIT） |
| `OBJ` | 交互物 | 燃气阀门位置，不可站上去，只能相邻交互 | 否 |

此外，地图标记中的 `P`（玩家起始位）映射为 COR，`G`（奶奶起始位）映射为 SMK，`ITEM`（面罩位）映射为 KIT。

#### 1.2 关键坐标配置

| 对象 | 坐标 (x, y) |
|---|---|
| 玩家初始位置 | (1, 4) |
| 王奶奶初始位置 | (8, 5) |
| 防烟面罩 | (4, 4) |
| 翻倒冰箱 | (3, 4) |
| 倒下储物架 | (7, 1) |
| 燃气阀门 | (8, 1) |

安全区判定函数 `inSafeZone`：x <= 1 且 y 在 3~5 之间（即左侧楼道的 3 格）。

#### 1.3 回合与行动力

| 参数 | 值 |
|---|---|
| 最大回合数 `MAX_ROUND` | 12 |
| 每回合行动力 `MAX_AP` | 4 |
| 初始回合 | 1 |

每回合结束时行动力重置为 4。当行动力归零后，系统会在 260ms 延迟后自动触发回合结算（`resolveEnvironment`）。玩家也可主动点击「回合结束」按钮提前结算。

### 二、开场决策：安全判断与 119 呼救

游戏启动后，所有行动卡和移动操作被锁定，必须先完成第一步决策。系统弹出决策卡「第一步应该做什么？」，提供 4 个选项：

| 选项 | 结果 | 具体效果 |
|---|---|---|
| 判断现场安全并呼叫 119（推荐） | 正确 | 消耗 1 行动力，设置 `hasCalled119 = true`，解锁全部后续玩法 |
| 直接冲进厨房 | 错误（保留弹窗） | 消耗 1 行动力，火场风险 +10，计入错误次数，重新弹出决策卡 |
| 回房间找贵重物品 | 严重错误 | 计入严重错误 +1，**直接游戏失败**，提示"火灾中不能返回取物" |
| 打开窗户大声喊 | 部分错误（保留弹窗） | 消耗 1 行动力，火场风险 +5，计入错误次数，重新弹出决策卡 |

**设计意图**：教育玩家火灾第一原则——先判断自身安全并呼叫专业救援。只有完成此步骤后，回合制行动系统才正式解锁。

### 三、战争迷雾与观察系统

#### 3.1 迷雾机制

地图初始只有左侧楼道区域（9 格：x 0~2, y 3~5）以及玩家周围半径 1 的范围是可见的。其余格子被深色迷雾覆盖（`fogDark` 色，alpha 0.76）。

迷雾格可以被点击，但点击行为等同于向该方向移动或观察。x >= 6 的迷雾格上还会叠加半透明烟雾图层，暗示右侧有烟区。

#### 3.2 视野揭示规则

| 触发条件 | 揭示范围 |
|---|---|
| 玩家移动后（无面罩） | 目标格周围曼哈顿距离 <= 1 |
| 玩家移动后（有面罩） | 目标格周围曼哈顿距离 <= 2 |
| 使用观察行动（无面罩） | 沿选定方向 2 格 |
| 使用观察行动（有面罩） | 沿选定方向 3 格 |
| 花花求助技能 | 玩家周围 3x3 全部揭示 |

观察遇到墙体即停止延伸。揭示过的格子永久可见（`revealed` 集合）。

#### 3.3 观察行动卡

消耗 1 行动力进入观察模式，然后点击玩家相邻的上/下/左/右方向格，沿该方向直线揭示 2~3 格视野。如果系统引导推荐的行动恰好是观察（`guidance.actionId === 'observe'`），则自动寻路到推荐方向并直接执行，省去手动选方向的一步。

观察执行后会触发发现检测（`checkDiscoveries`），如果新揭示的区域内有面罩、阀门或王奶奶，会更新对应的发现状态并给出花花提示。

### 四、移动系统

#### 4.1 基本移动

玩家通过点击相邻格（曼哈顿距离 = 1）来移动，每次消耗 1 行动力。移动支持四个方向：上、下、左、右。

移动时有平滑的 Tween 动画（150ms，Sine.easeOut），角色精灵会根据移动方向播放对应的行走动画（walk_up/down/left/right）。

#### 4.2 移动中的特殊触发

| 条件 | 效果 |
|---|---|
| 进入入口区域（x=2, y=3~5）且首次进入 | 触发厨房火焰环境音效（`startFireLoop`），花花提示"这里有烟了" |
| 进入烟雾格（SMK/HSMK） | 精灵切换为匍匐爬行动画（`player_crawl`），提示低姿前进 |
| x >= 5 且首次 | 触发"听到王奶奶咳嗽"剧情，设置 `heardGrandma = true` |
| 站到面罩格上 | 花花提示使用"现场处理"拾取面罩 |
| 点击未揭示的不可通行格 | 揭示该格并提示"这个方向暂时不能通过"，不消耗行动力 |
| 点击明火格 | **消耗 1 行动力 + 火场风险 +5**，计入惩罚反馈，揭示该格 |

#### 4.3 移动限制

| 阻挡类型 | 提示 |
|---|---|
| 墙体 | "这里是墙体或不可进入区域" |
| 火焰 | "明火格不能通行。绕开它" |
| 障碍（冰箱/储物架） | "这里被障碍挡住了。靠近后使用现场处理清障" |
| 阀门格 | "阀门需要相邻交互，不能站到阀门格" |
| 王奶奶所在格 | "不要站到王奶奶身上。停在相邻格，用沟通护送安抚她" |

#### 4.4 玩家轨迹记录

系统维护一个最多 4 步的 `playerTrail` 队列，记录玩家经过的位置。这个轨迹在护送模式中用于决定王奶奶的跟随目标（她会尝试移动到玩家的上一格位置）。

### 五、现场处理行动卡

"现场处理"是一张多功能行动卡，根据玩家当前相邻的可交互对象自动切换行为。

#### 5.1 拾取防烟面罩

- **触发条件**：玩家站在面罩格 (4,4) 上，且尚未持有面罩
- **消耗**：1 行动力
- **效果**：`hasMask = true`；后续移动揭示范围从半径 1 扩大到 2；观察距离从 2 格扩大到 3 格
- **拒绝选项**：不消耗行动力，但后续护送烟雾伤害更高

#### 5.2 清除障碍

两个障碍物分别是翻倒的冰箱 (3,4) 和倒下的储物架 (7,1)。

| 参数 | 冰箱 | 储物架 |
|---|---|---|
| 触发条件 | 玩家相邻且未清除 | 玩家相邻且未清除 |
| 消耗 | 2 行动力 | 2 行动力 |
| 清除后效果 | `clearedFridge = true`，该格从 OBS 变为 KIT（可通行） | `clearedShelf = true`，该格从 OBS 变为 KIT，解锁阀门交互 |
| 拒绝选项 | "绕路也可以" | "放弃阀门先救人"（合理选择） |

#### 5.3 关闭燃气阀门

- **触发条件**：玩家相邻阀门格 (8,1)，且储物架已清除，且阀门未关闭
- **消耗**：1 行动力
- **效果**：`gasClosed = true`；火场风险 -15；火焰扩散频率从每 2 回合一次降为每 3 回合一次；每次扩散的风险增量从 +10 降为 +5
- **锁定条件**（以下情况「关闭燃气」选项被禁用）：
  - 阀门已关闭
  - 储物架未清除（"先清障才可能接近"）
  - 火场风险 >= 80（"风险过高，不建议再靠近"）
  - 阀门相邻格有明火（"附近有明火，不要靠近"）

### 六、沟通护送行动卡

"沟通护送"同样是一张条件分支行动卡，根据玩家与王奶奶的状态自动呈现不同子操作。

#### 6.1 安抚沟通

- **触发条件**：玩家与王奶奶相邻（曼哈顿距离 <= 1），且王奶奶未被安抚
- **消耗**：1 行动力
- **决策卡选项**：

| 选项 | 效果 |
|---|---|
| 蹲下并慢慢说明身份（推荐） | `isCalmed = true`；恐慌值 -30；信任度 +30；标记 `foundGrandma = true` |
| 大声催促她快走（错误） | 恐慌值 +20；消耗 1 行动力；计入错误 |
| 直接拉她离开（错误） | 恐慌值 +30；信任度 -10；消耗 1 行动力；计入错误 |
| 让她自己找出口（错误） | 消耗 1 行动力；计入错误 |

**设计意图**：教育面对恐慌老人时应"蹲下、慢说、说明身份"，而非强拉硬拽。

#### 6.2 给王奶奶面罩

- **触发条件**：玩家与王奶奶相邻，已安抚，持有面罩，且王奶奶尚未戴面罩
- **消耗**：1 行动力
- **效果**：`grandmaMasked = true`；信任度 +10
- **机制影响**：王奶奶每回合受到的烟雾伤害大幅降低（SMK 从 5 降为 1，HSMK 从 8 降为 2）
- **拒绝选项**：不消耗行动力，但烟雾伤害维持高位

#### 6.3 引导撤离

- **触发条件**：玩家与王奶奶相邻，已安抚，信任度 >= 40，且护送模式未启动
- **消耗**：1 行动力
- **效果**：`escortMode = true`，王奶奶进入跟随状态
- **拒绝选项**：返回观察模式，重新规划路线

#### 6.4 等待跟随（护送模式中）

- **触发条件**：护送模式已激活
- **消耗**：1 行动力
- **效果**：如果玩家与王奶奶距离 > 1，王奶奶尝试移动到玩家轨迹的最后一格（前提是该格安全）；恐慌值 -5

### 七、王奶奶 AI 系统

#### 7.1 属性状态

| 属性 | 初始值 | 范围 | 说明 |
|---|---|---|---|
| 生命值 `grandmaHp` | 100 | 0~100 | 降至 0 则游戏失败 |
| 恐慌值 `panic` | 60 | 0~100 | 影响情绪表现 |
| 信任度 `trust` | 20 | 0~100 | >= 40 才能启动护送 |
| 已安抚 `isCalmed` | false | — | 正确安抚后变为 true |
| 已戴面罩 `grandmaMasked` | false | — | 给面罩后变为 true |
| 护送中 `escortMode` | false | — | 引导撤离后变为 true |

#### 7.2 恐慌移动（未安抚时）

当王奶奶未被安抚（`isCalmed = false`）且不在护送模式时，她会在每回合环境结算时随机移动一格。

**恐慌脚本**（`PANIC_SCRIPT`）定义了特定回合的预定移动目标：

| 回合 | 目标坐标 |
|---|---|
| 第 2 回合 | (7, 5) |
| 第 4 回合 | (7, 6) |
| 第 6 回合 | (8, 6) |

脚本未命中时，执行随机恐慌移动：从四方向相邻格中筛选合法位置（x 在 7~9，y 在 4~6 范围内，不能是墙体/火焰/障碍/阀门，不能与玩家重叠），优先选择远离浓烟且靠近玩家的方向。

如果移动后王奶奶进入已揭示区域，则自动标记 `foundGrandma = true`。

#### 7.3 跟随移动（护送模式中）

护送模式下，每次玩家移动后，系统检查玩家 **上一格位置** 是否对王奶奶安全（不是墙体/火焰/障碍/阀门，不与玩家重叠）。如果安全，王奶奶移动到该位置；否则花花提示"王奶奶不愿经过危险格"。

#### 7.4 烟雾伤害（回合结算时）

每回合结束时，如果王奶奶不在安全区，根据她所在格类型计算伤害：

| 格子类型 | 无面罩伤害 | 有面罩伤害 |
|---|---|---|
| SMK（低浓度烟雾） | 5 | 1 |
| HSMK（高浓度烟雾） | 8 | 2 |
| 其他非烟雾格 | 0 | 0 |

#### 7.5 可见性与表现

王奶奶的可见性遵循以下规则：当玩家与王奶奶的曼哈顿距离 <= 2 时，即使迷雾未揭示，王奶奶也会自动显示（`isGrandmaVisible` 返回 true）。此外，发现检测（`checkDiscoveries`）会在 **每次移动和每次观察后** 都运行——如果新揭示的区域包含面罩、阀门或王奶奶，花花会给出对应的发现提示消息。

| 条件 | 精灵 | 动画 | 状态标签 |
|---|---|---|---|
| 未发现且不可见 | 不渲染 | — | — |
| 听到但未找到 | 显示"咳咳……"气泡 | — | — |
| 恐慌（未安抚） | `grandma_idle` | `grandma_panic`（8fps） | "恐慌"（红色底） |
| 已安抚 | `grandma_calm` | 静态 | "已安抚"（绿色底） |
| 护送中（无面罩） | `grandma_walk_1/2` | `grandma_walk`（5fps） | — |
| 护送中（有面罩） | `grandma_masked_walk_1/2` | `grandma_masked_walk`（5fps） | "已防护" |

王奶奶头顶还有一个迷你生命条，颜色根据 HP 变化：>65 绿色，35~65 黄色，<=35 橙色。

### 八、火焰扩散与风险系统

#### 8.1 火焰扩散序列

火焰按照预定义的 `FIRE_SEQUENCE` 顺序逐格扩展（共 5 格）。**当 5 个序列位置全部消耗后，不再有新的火焰扩散**——这意味着如果玩家在火焰序列走完之前存活，后续回合的火焰压力将不再增加。

| 序号 | 坐标 | 说明 |
|---|---|---|
| 1 | (7, 4) | 厨房中部 |
| 2 | (7, 3) | 向上蔓延 |
| 3 | (6, 4) | 向左蔓延 |
| 4 | (9, 5) | 右侧远端 |
| 5 | (6, 5) | 向下蔓延 |

#### 8.2 扩散频率与条件

| 条件 | 频率 |
|---|---|
| 阀门未关闭 | 每 2 回合扩散一次（回合数 % 2 === 0） |
| 阀门已关闭 | 每 3 回合扩散一次（回合数 % 3 === 0） |

扩散只在回合结算（`resolveEnvironment`）时发生。

#### 8.3 扩散前的预警机制

当即将发生火焰扩散时，系统会先进入约 900ms 的预警阶段：

- 目标格上显示闪烁的"火势预警"标签（橙色高亮框，220ms 呼吸闪烁）
- 如果王奶奶也会恐慌移动，同时显示"烟雾扰动"预警（黄色）
- 花花提示"烟雾和火势有变化迹象"
- 预警结束后才实际执行火焰扩展和王奶奶恐慌移动

**状态锁定**：在整个预警和结算期间（约 900ms），`resolvingEnvironment` 标志位被设为 true，此时所有玩家操作被禁止——包括移动、行动卡、回合结束按钮均不可用，直到环境变化完全结算完毕。

#### 8.4 扩散效果

- 目标格变为永久火焰（加入 `dynamicFire` 集合，覆盖原始格子类型）
- 自动揭示该格
- 触发烟雾爆发特效（`smokeBurstTiles`）
- 火场风险增加：阀门未关 +10，阀门已关 +5
- 镜头震动（120ms，强度 0.002）
- 播放烟雾扩散音效

#### 8.5 火焰格保护

火焰不会扩展到玩家或王奶奶当前所在的格子。

#### 8.6 火场风险值

| 事件 | 变化 |
|---|---|
| 初始值 | 30 |
| 尝试穿越火焰格 | +5 |
| 错误行动 `applyWrongAction` | +5 或 +10（取决于参数） |
| 火焰扩散（阀门未关） | +10 |
| 火焰扩散（阀门已关） | +5 |
| 关闭阀门 | -15 |

风险值钳制在 0~100。达到 100 时游戏失败（"火势已不可安全接近"）。

### 九、花花助手系统

#### 9.1 助手面板

花花（宠物狗角色）常驻于右侧面板，包含以下信息区：

- **头像**：根据心情切换贴图（normal / hint / encourage / relieved）
- **标题栏**：显示当前模式（如"当前目标"、"花花指引"）
- **对话气泡**：动态更新的提示文本
- **目标清单**：用勾选格式展示 6 项任务进度
- **道具状态栏**：面罩、阀门、风险提示的实时状态

#### 9.2 心情与反馈动画

| 心情 | 头像 | 特效 |
|---|---|---|
| `normal` | `huahua_normal` | 无 |
| `hint` / `warn` | `huahua_hint` | 左右抖动 + 汗滴下落 |
| `encourage` / `relieved` | `huahua_encourage` / `huahua_relieved` | 爱心上浮 |

#### 9.3 智能引导系统

`getCurrentGuidance()` 函数按优先级返回当前最优行动建议：

| 优先级 | 条件 | 推荐行动 | 目标 |
|---|---|---|---|
| 1 | 可安抚王奶奶 | care | 王奶奶位置 |
| 2 | 可给王奶奶面罩 | care | 王奶奶位置 |
| 3 | 可启动护送 | care | 王奶奶位置 |
| 4 | 护送中 | move | 安全区方向的下一步 |
| 5 | 可拾取面罩 | scene | 面罩位置 |
| 6 | 相邻有障碍 | scene | 障碍物位置 |
| 7 | 可关闭阀门 | scene | 阀门位置 |
| 8 | 已发现面罩但未持有 | move | 面罩位置 |
| 9 | 已发现/可见王奶奶 | move | 王奶奶最近可达邻格 |
| 10 | 前方有未揭示区域 | observe | 前方推荐格 |
| 11 | 默认 | move | 前方推荐格 |

引导在地图上以高亮框 + 连线 + 气泡提示的形式呈现，推荐格有 760ms 呼吸脉冲动画。

#### 9.4 花花求助技能

- **消耗**：1 行动力
- **冷却**：3 回合（`huahuaCooldown`，每回合减 1）
- **效果**：
  1. 瞬间揭示玩家周围 3x3 区域
  2. 使用 BFS 寻路，计算从当前位置到目标的最优路径（护送中→安全区，否则→王奶奶）
  3. 路径以绿色发光圆点序列显示在地图上（每个点有 600ms+ 的脉冲动画）

#### 9.5 反馈浮层

每次花花提示更新时，会在地图下方中央显示一条浮动反馈条（最大显示 34 字符），2.6 秒后自动消失（最后 0.5 秒渐隐），颜色根据心情区分：鼓励/安心为浅绿底，提示/警告为浅红底。

### 十、邻居协助系统

呼叫 119 后，一个邻居 NPC 出现在安全区 (0, 4) 位置，头顶气泡显示"我去楼道接应119"。

- **触发方式**：点击 (0, 4) 格子
- **消耗**：0 行动力（免费）
- **效果**：`neighborAssigned = true`，花花提示"邻居去楼道接应119了"，邻居精灵和气泡消失
- **分数加成**：+3 分

### 十一、胜利与失败条件

#### 11.1 胜利条件

同时满足以下三个条件时触发胜利：

1. 护送模式已启动（`escortMode = true`）
2. 玩家在安全区内（`inSafeZone(player)`）
3. 王奶奶在安全区内（`inSafeZone(grandma)`）

**检测时机**：胜利条件仅在「玩家移动后」和「等待跟随行动后」两个时间点检查，不会在其他行动卡执行后或回合结束时触发。这意味着玩家必须通过主动移动进入安全区才能获胜。

胜利后弹出结算卡，显示分数和评级，并提供"查看知识卡"和"重新开始"两个选项。

#### 11.2 失败条件

| 触发条件 | 失败提示 |
|---|---|
| 王奶奶生命值降至 0 | "烟雾里停留太久了。下次要更快判断路线" |
| 火场风险达到 100 | "火势已经不可安全接近。现实中应撤离等待消防救援" |
| 回合数超过 12 | "救援窗口已经错过。下次减少无效移动" |
| 选择"回房间找贵重物品" | "火灾中不能返回取物。生命安全永远比物品更重要" |

### 十三、知识卡系统

胜利或失败后，玩家可以选择查看「火场协助知识卡」，内容为四条核心安全知识：

1. 先判断自身安全并呼叫 119，不盲目深入浓烟
2. 浓烟中低姿移动，看不清时先观察，不穿越明火
3. 面对恐慌老人，蹲下、慢说、说明身份，再引导撤离
4. 面罩和阀门是风险管理手段，但撤离生命优先

知识卡和结算卡之间可以自由切换。

### 十四、决策卡（Modal）交互系统

所有需要玩家做出选择的关键节点都通过统一的决策卡系统呈现（`showDecisionCard`），其设计特点如下：

- **遮罩层**：半透明暗黄色覆盖全屏，阻止其他操作
- **面板**：860px 宽的圆角卡片，自动测量正文高度以避免重叠
- **选项卡**：每个选项显示为 A/B/C/D 标签 + 行动说明，**不提前透露正确答案**（所有非锁定选项外观一致，奶油白底色）
- **锁定选项**：条件不满足时显示"锁定"标签，点击时弹出提示
- **保留弹窗**：部分错误选项设置 `keepOpen = true`，选择后不关闭弹窗，允许重新选择
- **持续弹窗**：结算卡设置 `persist = true`，必须通过选项按钮退出
- **交互反馈**：Hover 时 1.5% 放大，点击时 4% 缩小后弹回

### 十五、行动卡面板交互设计

### 十九、寻路系统

`findPath` 函数实现了标准的 BFS（广度优先搜索）寻路算法，用于：

1. **智能引导目标**：计算玩家到当前推荐目标的最短路径
2. **护送路径**：计算从当前位置到安全区的最短路径
3. **花花求助路径**：计算并显示完整路径点序列
4. **王奶奶最佳邻格**：找到从玩家到王奶奶最近的可达相邻格

寻路中的可通行判定使用 `isWalkableForPlayer`，排除墙体、火焰、障碍、阀门格，可选排除王奶奶所在格。

### 二十、面罩道具系统

防烟面罩是关卡一的核心道具，影响多个子系统：

| 阶段 | 状态标记 | 影响 |
|---|---|---|
| 未发现 | `foundMask = false` | 底部道具槽显示"未发现"，图片 28% 透明度 |
| 已发现未拾取 | `foundMask = true, hasMask = false` | 道具槽"去拾取"，面罩格有黄色脉冲光晕，每 3 回合花花提醒 |
| 已持有 | `hasMask = true` | 移动视野半径 +1，观察距离 +1，道具槽"可使用" |
| 已给王奶奶 | `grandmaMasked = true` | 王奶奶烟雾伤害大幅降低，道具槽"已给王奶奶" |

面罩提醒机制：如果已发现面罩但未拾取，每隔 3 回合花花会提醒"防烟面罩已经发现，还没拾取"。

### 二十一、图层渲染架构

场景使用 10 个 Phaser Container 分层管理，从底到顶依次为：

| 图层 | 用途 |
|---|---|
| `staticLayer` | 背景底色、HUD、助手面板等不变元素 |
| `gridLayer` | 网格地砖、厨房装饰、图例 |
| `objectLayer` | 烟雾/火焰特效、可交互物品、预警标签 |
| `actorLayer` | 玩家、王奶奶、邻居精灵及状态标签 |
| `fogLayer` | 战争迷雾覆盖层 |
| `guidanceLayer` | 引导高亮、路径提示、反馈浮层 |
| `actionLayer` | 底部行动卡面板 |
| `utilityLayer` | 回合结束按钮 |
| `uiLayer` | 顶层 UI（不受遮罩影响） |
| `modalLayer` | 决策卡弹窗 |

每回合或状态变化时，`refreshScene()` 会按顺序重绘：网格 → 物体 → 角色 → 迷雾 → 引导 → HUD → 行动卡 → 功能按钮。
