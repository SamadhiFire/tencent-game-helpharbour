# 《红花救援队》关卡二正式策划案：广场黄金四分钟

## 一、文档定位与全局口径

本文件用于定义《红花救援队》正式关卡二的教学目标、玩法规则、数值模型、流程、评分与开发实现边界。关卡二承接关卡一的网格和行动卡基础，但将体验重点从“探索护送”切换为“急救流程判断 + 现场分工 + CPR节奏 + AED等待”。

### 1. 全局设计原则

| 维度 | 统一口径 |
| --- | --- |
| 游戏模式 | 只设置“标准模式”，不做高压惩罚模式或复杂操作模式 |
| 基础玩法 | 网格救援 + 行动卡判断 + 关键节点小游戏 + AI复盘 |
| 失败处理 | 允许立即重试，并给出错误原因与正确流程复盘 |
| 公益优先级 | 正确急救流程高于操作刺激，错误急救动作不能成为捷径 |
| AI助手定位 | “花花”提供提示、反馈、复盘和知识卡，不自动完成核心操作 |
| 全龄体验 | 大按钮、清晰文字、明确图标、回合制推进，不因现实思考时间惩罚玩家 |

### 2. 本关的回合口径

关卡二分为两个操作阶段：

| 阶段 | 回合与资源 |
| --- | --- |
| 现场判断阶段 | 沿用网格AP，每回合4 AP，移动、检查、指派路人都消耗AP |
| CPR维持阶段 | 每轮CPR QTE等于1个救援回合，玩家不能离开老人身边；场面管理改用“指挥点” |

“指挥点”用于模拟玩家在持续急救中对旁人的简短指令，而不是让玩家停止CPR到处移动。

| 项目 | 规则 |
| --- | --- |
| CPR阶段每回合指挥点 | 1点 |
| 指挥点可做什么 | 指挥群众让路、提醒路人B继续取AED、请求路人A维持通道 |
| 指挥点不能做什么 | 不能替代CPR，不能让玩家离开老人身边 |
| 设计目的 | 保持急救流程正确，同时保留现场管理的游戏性 |

## 二、关卡概览

| 项目 | 内容 |
| --- | --- |
| 关卡编号 | 正式关卡二 |
| 关卡名称 | 广场黄金四分钟 |
| 推荐时长 | 3-4分钟 |
| 推荐有效回合 | 10-12回合 |
| 核心目标 | 在救护车到来前维持倒地老人的生命稳定状态 |
| 教学主题 | 心脏骤停识别、呼救、AED、CPR |
| 玩法关键词 | 步骤判断、路人指派、CPR QTE、AED跑腿、群众疏散 |
| 本关在全游戏中的职责 | 强化“先判断、再呼救、持续急救、等待专业救援”的完整急救链 |

## 三、教学目标与体验目标

### 1. 公益教学目标

玩家完成本关后，应形成以下认知：

| 教学点 | 玩家需要学会的内容 |
| --- | --- |
| 判断现场安全 | 靠近倒地者前先确认环境安全 |
| 检查反应 | 轻拍双肩并呼唤，不用力摇晃 |
| 及时呼救 | 指派明确对象拨打120并寻找AED |
| 判断呼吸 | 无正常呼吸时立即开始CPR |
| 持续按压 | AED到达前持续进行胸外按压，不围观等待 |
| AED使用 | 打开设备，按语音提示贴片、离身、分析、除颤并继续CPR |

### 2. 游戏体验目标

本关希望玩家感受到：

- 急救流程判断的清晰顺序感。
- CPR节奏带来的参与感，但不制造过度惩罚。
- 看着路人B取AED往返时的时间压力。
- 群众干扰造成的现场管理压力。
- AED送达后的情绪释放。

## 四、核心玩法循环

本关的核心循环是“急救步骤正确性 + 持续维持 + 现场协作”。

1. 玩家靠近倒地老人，完成环境和意识判断。
2. 玩家明确指派路人A拨打120，路人B寻找AED。
3. 玩家判断呼吸后进入CPR节奏QTE。
4. 路人B在网格中真实移动，玩家一边做CPR一边用指挥点维持通道。
5. AED到达后，玩家按步骤卡正确使用AED。
6. 救护车到达后结算，并生成复盘与知识卡。

## 五、场景与地图设计

### 1. 地图尺寸

本关地图为 **10x7网格**，横向10格，纵向7格。

采用10x7的原因：

- 横向距离足以表现“倒地老人现场 - AED柜”的等待感。
- 路人B单程约4回合，往返约8回合，适合形成CPR维持压力。
- 玩家大部分时间留在老人身边，地图主要服务于“可视化等待”和“现场协作”。

### 2. 空间分区

| 区域 | 位置 | 功能 |
| --- | --- | --- |
| 救援核心区 | 左侧3-4列 | 倒地老人、玩家、路人A、路人B初始位置 |
| 人群通道区 | 中间3列 | 群众占格与AED通道管理 |
| AED区域 | 右侧2-3列 | AED柜与路人B跑腿目标 |
| 救护车入口 | 地图边缘 | 拨打120后出现倒计时标志 |

### 3. 主要元素

| 元素 | 设计说明 |
| --- | --- |
| 倒地老人 | 核心救助对象，位于左侧救援核心区 |
| 玩家 | 靠近老人后基本固定在老人身旁 |
| 路人A | 可被指派拨打120，并在CPR阶段协助维持通道 |
| 路人B | 可被指派寻找AED，之后在网格上真实移动 |
| AED柜 | 位于右侧边缘，距离老人约7-8格 |
| 围观群众 | 随机占据通道或老人周围格子，喊出错误建议 |
| 救护车标志 | 拨打120后出现，显示救援回合倒计时 |

## 六、机制与数值模型

### 1. 初始数值

本关使用“生命稳定度”而不是直接写实医疗数值，避免让玩家误解为真实医学模拟。

| 项目 | 数值 |
| --- | --- |
| 老人初始生命稳定度 | 100 |
| 救护车到达时间 | 拨打120后12回合 |
| 未开始CPR时消耗 | 每回合 -10生命稳定度 |
| CPR较差时消耗 | 每回合净 -5生命稳定度 |
| CPR一般时消耗 | 每回合净 -3生命稳定度 |
| CPR优秀时消耗 | 每回合净 -1生命稳定度 |
| 连续2轮优秀 | 额外 +2生命稳定度 |
| AED正确使用 | 生命稳定度 +20，后续每回合基础消耗降低2点 |
| 路人B移动速度 | 每回合2格 |
| AED柜距离 | 单程7-8格，往返约8回合 |
| 群众堵路惩罚 | 路人B该回合少移动1格 |
| 失败条件 | 生命稳定度归零 |

### 2. 现场判断阶段AP

| 行为 | AP消耗 |
| --- | --- |
| 移动1格 | 1 AP |
| 判断环境 | 1 AP |
| 检查意识 | 1 AP |
| 指派路人A拨打120 | 1 AP |
| 指派路人B寻找AED | 1 AP |
| 判断呼吸 | 1 AP |
| 错误动作 | 消耗1 AP并触发惩罚 |

玩家应在2个回合内完成“判断环境 - 检查意识 - 呼救/AED - 判断呼吸”，否则老人生命稳定度会快速下降。

### 3. CPR阶段QTE

每轮CPR QTE对应1个救援回合，不使用现实倒计时扣血。玩家看弹窗、理解信息、选择下一步时不会被现实时间惩罚。

| QTE表现 | 判定 | 生命稳定度变化 |
| --- | --- | --- |
| 成功率 >= 70% | CPR优秀 | 本回合净 -1 |
| 成功率40%-69% | CPR一般 | 本回合净 -3 |
| 成功率 < 40% | CPR较差 | 本回合净 -5，花花提示调整节奏 |
| 连续2轮优秀 | 稳定状态 | 额外 +2 |

QTE设计边界：

- QTE只表达“保持节奏”的参与感，不替代真实急救培训。
- 失败不会立刻致命，避免公益游戏变成手速惩罚。
- 花花要提示“现实中应接受专业培训，按当地急救指导执行”。

### 4. AED跑腿与群众占格

路人B被指派后开始移动：

- 每回合向AED柜移动2格。
- 到达AED柜后自动取得AED。
- 返回时仍每回合移动2格。
- 路人B进入老人相邻格后触发AED交接。

围观群众机制：

| 项目 | 规则 |
| --- | --- |
| 出现频率 | 每1-2回合最多出现1名 |
| 同屏上限 | 最多3名群众占格 |
| 出现位置 | 老人周围或中部通道空格 |
| 错误建议 | 以气泡形式显示，不弹出“忽略”按钮 |
| 堵路影响 | 如果占据路人B路径，路人B该回合少走1格 |
| 处理方式 | CPR阶段消耗1指挥点，指派路人A或大声请求让出通道 |

这样可以保留现场管理玩法，同时不破坏“玩家应持续CPR”的公益原则。

## 七、标准通关流程

### 阶段一：靠近倒地老人并判断环境

触发事件：

围观群众：“他突然倒下了！快来帮忙！”

花花提示：

“先确认现场是否安全，再检查他的反应。”

行动卡：

| 选项 | 结果 |
| --- | --- |
| 判断现场环境 | 正确，进入下一步 |
| 立刻扶起老人 | 错误，生命稳定度 -15 |
| 给老人喝水 | 错误，生命稳定度 -10 |
| 掐人中 | 错误，生命稳定度 -8 |

### 阶段二：检查意识

行动卡：

| 选项 | 结果 |
| --- | --- |
| 轻拍双肩并呼唤 | 正确，老人无反应 |
| 用力摇晃 | 错误，生命稳定度 -10 |
| 直接做胸外按压 | 流程不完整，花花提示先呼救并判断呼吸 |
| 等他自己醒来 | 错误，每回合继续 -10 |

正确结果：

花花：“没有反应，需要立即呼救，并请人寻找AED。”

### 阶段三：明确指派路人

玩家点击路人A，弹出指令：

| 选项 | 结果 |
| --- | --- |
| 请拨打120 | 正确，救护车倒计时启动 |
| 请帮我拍视频 | 错误，评分下降 |
| 请把老人扶起来 | 错误，生命稳定度 -15 |
| 请散开一点 | 不完整，需要先拨打120 |

玩家点击路人B，弹出指令：

| 选项 | 结果 |
| --- | --- |
| 请寻找AED | 正确，路人B开始向AED柜移动 |
| 请给老人喝水 | 错误，生命稳定度 -10 |
| 请围住现场 | 错误，群众占格概率提高 |
| 请不要动 | 错误，AED不会送达 |

### 阶段四：判断呼吸并开始CPR

行动卡：

| 选项 | 结果 |
| --- | --- |
| 观察胸廓起伏并判断呼吸 | 正确，确认无正常呼吸 |
| 直接喂水 | 错误，生命稳定度 -10 |
| 把老人扶坐起来 | 错误，生命稳定度 -15 |
| 按摩太阳穴 | 错误，生命稳定度 -8 |

系统提示：

“老人无正常呼吸，立即开始CPR。让路人继续取AED，救护车正在路上。”

### 阶段五：CPR维持与现场指挥

进入CPR界面后，画面同屏显示：

- 左侧或上方：网格地图，可看到路人B跑向AED并返回。
- 主区域：CPR节奏QTE。
- 辅助区域：生命稳定度、救护车到达回合、AED进度、当前指挥点。

每回合流程：

1. 玩家完成一轮CPR QTE。
2. 系统结算生命稳定度。
3. 路人B移动2格。
4. 群众可能占格或堵路。
5. 玩家可消耗1指挥点处理一个现场问题。

指挥点选项：

| 选项 | 效果 |
| --- | --- |
| 请大家让出AED通道 | 清除路人B路径上的1名群众 |
| 请路人A维持现场秩序 | 下一回合群众出现概率降低 |
| 提醒路人B继续取AED | 如果路人B被堵，恢复1格移动 |
| 保持CPR不分心 | 不处理现场问题，获得少量QTE容错提示 |

### 阶段六：AED交接与使用

当路人B带着AED进入老人相邻格，触发“网格交接”动画：

- 画面短暂聚焦路人B。
- 路人B将AED递给玩家。
- 花花提示：“AED到了，打开设备，按语音提示操作。”

AED步骤卡：

1. 打开AED。
2. 暴露并保持胸部干燥。
3. 按图示贴上电极片。
4. 等待设备分析。
5. 提醒所有人离开身体。
6. 按设备提示除颤。
7. 立即继续CPR。

判定规则：

| 情况 | 结果 |
| --- | --- |
| 顺序正确 | 生命稳定度 +20，后续消耗降低 |
| 漏掉“所有人离开身体” | 不允许除颤，花花提示纠正 |
| 贴片顺序错误 | 当回合净 -5，纠正后重试 |
| 未派人找AED | 只能持续CPR，通关可行但评分上限降低 |

### 阶段七：救护车到达与结算

救护车倒计时结束后进入结算。

如果老人生命稳定度大于0，关卡完成。

结算展示：

- 是否判断环境。
- 是否正确检查意识。
- 是否及时拨打120。
- 是否寻找AED。
- CPR平均准确率。
- AED是否正确使用。
- 是否处理群众堵路。
- 最终生命稳定度。

## 八、错误处理与教学反馈

| 错误操作 | 游戏后果 | 教学反馈 |
| --- | --- | --- |
| 直接扶起老人 | 生命稳定度 -15，评分下降 | 倒地无反应者不能随意扶起 |
| 喂水 | 生命稳定度 -10，评分下降 | 无意识或呼吸异常者不要喂水 |
| 掐人中 | 生命稳定度 -8，评分下降 | 不应以民间偏方替代急救流程 |
| 未拨打120 | 救护车不启动，评分上限大幅降低 | 急救必须尽快呼叫专业救援 |
| 未寻找AED | AED不会到达，通关难度增加 | AED越早使用越有利 |
| CPR QTE较差 | 每回合净 -5 | 保持稳定节奏，跟随提示调整 |
| AED步骤错误 | 当回合净 -5，纠正后重试 | 按设备语音和图示操作 |
| 未疏散堵路群众 | AED送达延迟 | 围观会影响救援通道 |

## 九、胜负条件与评分

### 1. 胜利条件

救护车到达时，老人生命稳定度大于0，且玩家完成基础呼救和CPR流程。

### 2. 失败条件

- 老人生命稳定度归零。
- 长时间未呼救且无法触发救护车到达。
- 连续选择严重错误操作导致复盘中断。

### 3. 评分结构

满分100分：

| 评分项 | 分值 | 说明 |
| --- | --- | --- |
| 急救流程 | 30 | 判断环境、检查意识、判断呼吸、及时CPR |
| 呼救与协作 | 20 | 指派120、寻找AED、维持通道 |
| CPR表现 | 20 | QTE平均准确率与连续稳定表现 |
| AED使用 | 20 | AED送达、步骤正确、离身提醒 |
| 现场管理 | 10 | 疏散群众、未听信错误建议 |

高分目标是“正确流程 + 稳定执行 + 及时协作”，不是单纯追求最快点击。

## 十、AI助手与复盘设计

### 1. 关卡中提示

花花提示遵循“三层提示”：

| 层级 | 触发 | 内容 |
| --- | --- | --- |
| 流程提示 | 首次进入急救步骤 | 提醒下一步判断，不直接给答案 |
| 纠错提示 | 玩家选择错误动作 | 说明错误风险，并引导回正确流程 |
| 节奏提示 | CPR QTE连续较差 | 提示稳定节奏，不用责备玩家 |
| 复盘提示 | 失败或通关后 | 总结关键流程和公益知识 |

### 2. AI生成内容边界

AI可生成：

- 围观群众的干扰台词。
- 路人A、路人B的协助反馈。
- 倒地老人背景设定。
- 玩家通关后的个性化复盘。
- 小红花知识卡。

AI不可生成：

- 与急救流程冲突的建议。
- 鼓励喂水、扶起、掐人中等错误行为的正向反馈。
- 声称游戏训练等同于真实急救资质的文案。

## 十一、开发实现需求

本关需要实现以下系统：

- 10x7网格地图。
- 现场判断阶段的4 AP回合系统。
- CPR阶段的QTE回合系统。
- CPR阶段指挥点系统。
- 行动卡选择与错误反馈。
- 路人A、路人B指派。
- 路人B向AED柜往返的网格移动。
- 群众占格、堵路、疏散。
- 生命稳定度模型。
- 救护车回合倒计时。
- CPR节奏QTE与网格地图同屏显示。
- AED交接动画。
- AED步骤卡顺序判定。
- 胜负判定与评分。
- AI提示、复盘和知识卡接口。

## 十二、开发阶段目标

本关作为第三阶段核心关卡，开发目标是验证：

- 现场判断流程是否清晰。
- 指派路人拨打120和寻找AED是否自然。
- CPR QTE是否有参与感但不过度惩罚。
- 路人B跑腿是否能有效制造等待压力。
- 指挥点是否解决“玩家不能离开CPR”的公益矛盾。
- 群众干扰是否形成游戏性，而不是纯随机惩罚。
- AED步骤卡是否准确、可读、可操作。
