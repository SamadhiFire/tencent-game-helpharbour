# 双关卡 UI 统一规范

> 适用范围：关卡一「烟雾厨房里的王奶奶」、关卡二「广场黄金四分钟」及后续公益救援关卡。  
> 参考来源：项目内 `tools/taste-skill/taste-skill-main`，采用“先统一交互骨架，再保留关卡机制差异”的改造方式。

## 1. 统一设计目标

双关卡 UI 必须让玩家形成同一套操作记忆：上方看状态，右侧看花花提示，下方选救援行动，弹窗做救援决策。

关卡机制可以差异化：关卡一强调烟雾、路径、安抚、护送；关卡二强调急救流程、CPR、AED、群众协作。但 UI 语言不允许像两个独立小游戏。

## 2. 画布与布局

- 基准画布：`1600 x 900`。
- 顶部 HUD：高度 `88px`，固定承载关卡名、行动点、核心生命/稳定指标、风险/阶段、回合/指挥信息。
- 主游戏区：`y=100-730`，承载网格、角色、路径、烟雾/广场环境。
- 底部行动区：`y=730-900`，承载救援行动卡和关卡专属仪表。
- 右侧花花助手：默认 `286 x 520`，位于右侧安全信息区，不遮挡网格核心目标。
- 工具按钮：右下纵向排列，作为“回合结束、重新开始、流程提示、行动日志”等辅助功能入口。

## 3. 颜色规范

主色采用深色救援指挥界面，不再使用浅色拼贴按钮。

- 深底色：`#071421` / `#101820`，用于整体背景、弹窗遮罩和底部操作区。
- 面板色：`#0D2032` / `#132A3D`，用于 HUD 指标、花花面板、行动卡底。
- 分割线：`#40576B`，用于 HUD、面板描边。
- 温暖文本：`#F6E5C8`，用于标题、重要数值、按钮主文字。
- 推荐色：`#F6C85F`，用于“建议行动”标签和推荐描边。
- 安全/成功：`#2FBF7A`，用于生命稳定、正确流程推进。
- 信息蓝：`#1F8FE5`，用于普通行动与观察类操作。
- 风险/错误：`#E14532`，用于高危行为和二次确认。

## 4. 字号规范

- 关卡标题：`26px`，加粗。
- HUD 数值：`22px`，加粗，尽量短文本，避免长句塞入顶部。
- HUD 标签：`13px`，加粗，低对比辅助。
- 行动卡标题：`21-24px`，加粗。
- 行动卡说明：`14-15px`。
- 决策卡标题：`28px`，加粗。
- 决策卡正文：`17px`，行距 `8px`。
- 花花主提示：`20px`，加粗，最多 2-3 行。

## 5. HUD 规范

HUD 使用统一组件 `TopHud`：

- 左侧：关卡序号 + 关卡标题 + 单字主题徽标。
- 中右侧：指标胶囊按重要性排列。
- 关卡一指标：行动点、王奶奶生命、情绪状态、火场风险、救援窗口。
- 关卡二指标：行动点、生命稳定度、急救阶段、现场指挥。
- 指标值只放结果，不写长解释；解释交给花花助手或目标清单。

## 6. 行动卡规范

行动卡使用统一组件 `ActionDock`：

- 卡片不是“答题按钮”，而是玩家当前可执行的救援行动。
- 每张卡必须包含：图标、行动名、代价、短说明。
- 推荐行动统一显示“建议”标签，使用黄色描边。
- 不可用行动不隐藏，降低透明度并保留原因反馈。
- 关卡一固定四类：观察、低姿前进、场景处理、关怀/护送。
- 关卡二保留机制差异：CPR/AED 状态仪表独立显示，右侧仍使用统一救援行动卡。

## 7. 决策弹窗规范

弹窗统一为 `DecisionOverlay`，标题固定为“救援决策卡”。

每个选项以卡片呈现，必须包含：

- 行动标签：建议行动 / 可选策略 / 高危行为 / 暂不可用。
- 行动代价：如 `1 AP`、`1 指挥点`、`步骤卡`、`不消耗AP`。
- 风险等级：风险低 / 需权衡 / 风险高 / 条件未满足。
- 结果预期：描述执行后对救援流程的影响。
- 花花反馈：告诉玩家执行后会获得怎样的引导或纠偏。

禁止把“正确/错误”作为按钮核心文案。脚本内部可以继续用 `recommended` 和 `danger` 判断，但界面上要转译为救援语义。

高危行为必须二次确认：第一次点击进入红色确认态，第二次点击才执行。这样既保留学习试错，也避免玩家误触导致挫败。

## 8. 花花助手规范

花花助手使用统一组件 `AssistantPanel`：

- 上部：花花头像和名字。
- 中部：当前提示，语言要短、具体、行动导向。
- 下部：目标清单、物品/救援状态。
- 花花不直接替玩家做选择，但要解释为什么推荐某一步。
- 错误反馈要公益教育化：说明风险和正确原则，避免只说“错了”。

## 9. 间距与形状

- 大面板圆角：`12-16px`。
- 行动卡圆角：`14px`。
- 内部标签圆角：`9-10px`。
- 卡片间距：`16-18px`。
- HUD 指标间距：`14px`。
- 底部行动区与主游戏区分割线固定在 `y=730`。

## 10. 实现约定

共享 UI 位于 `src/game/ui/RescueUi.js`：

- `TopHud`：`drawTopHud`
- `ActionDock`：`drawActionDock`
- `DecisionOverlay`：`drawDecisionOverlay`
- `AssistantPanel`：`drawAssistantPanel`
- `UtilityButtons`：`drawUtilityButtons`

关卡 Scene 只负责传入状态、文案和回调，不直接复制 UI 样式。新增关卡优先复用这些组件，再根据机制添加局部仪表。
