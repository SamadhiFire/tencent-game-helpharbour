## 三、AI Prompt 模板

以下 Prompt 需要预先设计并写入系统，开发时由代码传入参数调用。

| 序号 | Prompt名称 | 用途 | 输入参数 | 优先级 |
|------|-----------|------|----------|--------|
| 50 | 王奶奶背景故事 | 关卡一开场生成 | `{npc_name}`, `{age}`, `{condition}` | P1 |
| 51 | 王奶奶情绪台词 | 关卡一动态对话 | `{恐慌值}`, `{信任值}`, `{玩家操作}` | P1 |
| 52 | 倒地老人背景 | 关卡二开场生成 | `{scene}`, `{condition}` | P1 |
| 53 | 围观群众干扰台词 | 关卡二占格群众气泡 | `{round}`, `{elder_hp}` | P1 |
| 54 | 赛后复盘文案 | 两关结算 | `{操作记录}`, `{评分}`, `{错误项}`, `{遗漏项}` | P0 |
| 55 | 小红花知识卡 | 两关通关 | `{关卡知识点}`, `{玩家错误类型}` | P0 |
| 56 | 花花引导台词 | 两关关键步骤 | `{当前步骤}`, `{NPC状态}`, `{回合数}` | P1 |

### Prompt 设计规范

- 每个 Prompt 需要包含系统指令（角色设定）+ 输出格式约束 + Few-shot 示例
- 急救流程、正确答案由系统预设模板控制，AI 不即兴生成急救建议
- 所有 AI 生成内容需通过关键词白名单过滤，防止偏离公益主题

## 四、技术资源

### 4.1 系统模块（需新建/更新）

| 模块 | 原方案 | 更新后 | 影响 |
|------|--------|--------|------|
| GridMapSystem | 网格+移动+障碍 | + **Fog of War** 子模块 | 关卡一浓烟视野 |
| TurnSystem | 回合+AP+扩散 | + **回合驱动数值公式** | 关卡二不再依赖现实时间 |
| NPCSystem | 生命+恐慌+信任 | 不变 | — |
| NPCMoveSystem | — | **新建**：NPC随机移动 + 网格物理移动 | 奶奶恐慌移动 + 路人B跑腿 |
| CrowdSystem | — | **新建**：群众占格+疏散+堵路检测 | 关卡二群众不再是弹窗 |
| QTESystem | CPR QTE | + **同屏显示**（QTE+网格） | 关卡二 CPR 时不切画面 |
| AccessibilitySystem | 标准/暖心模式 | **移除**（统一标准模式） | 降低开发复杂度 |

## 五、资源优先级总结

```
P0（MVP 不可缺）—— 8项美术 + 5项音频 + 3项Prompt
  ├─ 玩家、王奶奶、倒地老人、路人A、路人B 角色
  ├─ 厨房8×8 + 广场7×7 网格地图
  ├─ 防烟面罩、燃气阀门、AED设备、火焰格、烟雾格 物件
  ├─ AP/HP/恐慌/信任 UI条、行动卡按钮、道具栏
  ├─ CPR QTE圆环 + 同屏布局 + AED步骤卡
  ├─ CPR命中/失误音
  └─ 复盘Prompt、知识卡Prompt

P1（体验完整）—— 10项美术 + 4项音频 + 4项Prompt
  ├─ 花花头像、围观群众（1个模板）、路人B跑步动画
  ├─ Fog of War遮罩、翻倒冰箱、倒下的储物架
  ├─ 安全区高亮、AED柜、救护车标志、小红花符号
  ├─ 回合数、花花提示框、群众气泡、结算面板、复盘面板
  ├─ 火势环境音、拾取音效、AED提示音、救护车/胜利音
  └─ 王奶奶故事/台词、老人背景、群众台词、花花引导 Prompt

P2（打磨加分）—— 剩余美术变体 + 全部音频 + 主菜单
  ├─ 围观群众 3-4个变体
  ├─ 知识卡图鉴、主菜单
  └─ 烟雾扩散/清障音效、群众背景音、花花语音
```

## 六、设计阶段建议

| 阶段 | 产出物 | 预计耗时 |
|------|--------|----------|
| 阶段一：灰盒 | 纯色网格地图 + 方块角色 + 基础AP/HP交互 | 无美术门槛，纯代码 |
| 阶段二：白盒 | 像素风格角色 + 地图底色 + Fog of War + CPR QTE | 美术开始介入 |
| 阶段三：润色 | 全部物件图标 + 音效 + AI Prompt 接入 + 结算UI | 前后端联调 |
| 阶段四：打磨 | 动画帧、群众变体、音效完整、主菜单 | 收尾 |

> **建议**：阶段一先用灰盒（彩色方块代替角色，纯色格代替地图）快速验证核心玩法逻辑。确认好玩后再投入美术资源。
